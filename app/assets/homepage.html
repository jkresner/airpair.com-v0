{ why we started airpair }

Jonathon Kresner
"I started Airpair to help developers."

I've ALWAYS loved coding product. When things run smooth & you're knocking out screens, life is grand. But, often teams lose their way. Products get delayed and instead of knowing if your market assumptions are right, you're stuck battling technology. I know because I was there in both of my first companies. I needed access to mobile experts and DB gurus, even though I consider myself a strong engineer.

Airpair is for companies that want to get back on track, or better, start off on the right foot by periodically getting expert help to deliver faster.

Maksim Ioffe
"Airpair is an evolution of the agile methodology."

I did a stint as a software consultant at Pivotal Labs, where pair programming is done 100% of the time. I loved both learning and teaching how to develop software. It was also there when I pair programmed remotely for the first time. I was blown away by how evolved the tools and infrastructure had become to enable this type of an experience.

I wanted to create a platform that would enable anyone and everyone to get access to top technical talent for remote pair programming and technical help.
